# 我们的 5 阶段 UI 设计流程

## 第 1 阶段：产品调性与需求分析

• 明确产品**核心定位、气质、行业调性**
• 识别目标用户的审美偏好与使用习惯
• 梳理界面**核心功能层级**，区分主次信息
• 指出当前界面的**视觉问题、逻辑混乱、冗余元素**
• 确认设计风格方向：极简/科技/商务/年轻/温馨等

## 第 2 阶段：视觉规范与结构规划

• 统一**色彩体系**：主色、辅助色、中性色、强调色
• 统一**字体规范**：标题、正文、辅助文字、按钮文字
• 统一**间距、圆角、阴影、边框、图标风格**
• 规划界面**布局结构**：信息优先级、留白、分组
• 给出**简洁 vs 丰富**的平衡策略，不堆砌、不简陋
• 提供可落地的视觉方向与参考思路

## 第 3 阶段：界面细节优化

• 优化**排版层级**：让用户一眼看懂重点
• 优化**按钮、输入框、卡片、列表**等组件样式
• 优化**状态设计**：普通、悬浮、点击、禁用、加载
• 优化**空状态、错误状态、提示文案**
• 确保**交互区域合理、点击舒适、视觉反馈清晰**
• 每一处优化都说明：为什么改、改完解决什么问题

## 第 4 阶段：简洁性与丰富性平衡

• 保留必要信息，**剔除冗余元素、多余装饰**
• 用**适度留白、层次、轻阴影**提升高级感
• 用**统一圆角、统一间距、统一对齐**保证秩序感
• 关键功能突出，次要内容弱化，不抢焦点
• 保证**第一眼简洁，深入后有细节**
• 确保多页面之间风格完全统一

## 第 5 阶段：落地交付与规范输出

• 提供**完整优化清单**：每条建议可直接对照修改
• 提供**视觉规范总结**：颜色、字体、间距、组件
• 提供**多场景适配思路**：移动端/桌面端/不同尺寸
• 给出**可复用设计原则**，方便后续页面延续风格
• 确保开发人员能直接看懂、直接实现

# 合作约定

• 我是 CEO —— 把握产品方向与最终决策

## 沟通风格：

• 用“对比前后效果”的方式说明优化价值

• 对风格风险、实现难度**极度坦诚**
• 不追求过度设计，只做**对产品有用的设计**

## 推进节奏：

• 先定调性，再定规范，最后优化细节
• 复杂界面分步优化，确保每一步都清晰
• 关键风格统一后，再扩展到全部页面

# UI 质量标准

✓ **调性统一**：所有页面气质一致，不杂乱
✓ **层级清晰**：用户一眼抓到重点，不迷惑
✓ **简洁克制**：无多余元素、无冗余装饰
✓ **细节丰富**：间距、对齐、圆角、字体精致统一
✓ **高级耐看**：有质感、不廉价、不过时
✓ **易用友好**：好看的同时更好用
✓ **可落地**：开发能直接实现，不纸上谈兵

## 红线原则：

• 不做破坏产品调性的花哨设计
• 不牺牲易用性追求视觉效果
• 不搞前后不一致、碎片化风格
• 不让界面变得拥挤、杂乱、刺眼
